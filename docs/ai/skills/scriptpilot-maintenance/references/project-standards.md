# 项目规范

## 产品边界

- ScriptPilot 是本机使用的 Windows 绿色便携版 Electron 应用。
- 不做账号体系、角色体系或公网服务。
- 目标体验参考青龙面板，但数据和运行时留在本机目录。

## 数据目录

所有应用数据必须在便携目录内：

```text
release/win-unpacked/app/data/
```

关键约束：

- 不把用户数据写到 AppData。
- 不覆盖 `release/win-unpacked/app/data/`。
- 不把 `app/data/` 打进 GitHub Release 附件。
- 测试必须使用隔离 `portableRoot` 或隔离 Electron `cwd`。

## 架构分层

```text
src/electron/             Electron main、preload、本机 API、renderer
src/main/bootstrap/       绿色路径和目录约束
src/main/app/             应用装配
src/main/shared/          CommandBus、QueryBus、错误和通用基础设施
src/main/modules/tasks/   任务聚合、命令、查询、JSON repository
src/main/modules/runs/    运行记录、脚本执行、日志读取
src/main/modules/qinglong/ 环境、配置、脚本、订阅、依赖服务
```

Renderer 是普通 HTML/CSS/JS，不使用 React。后端是 ESM Node 模块。

## 重构原则

- 优先保持用户数据不丢失；内部模块 API、函数签名、UI 结构可以为了清晰度直接重构，不为旧内部写法保留兼容层。
- 重构必须降低复杂度：减少重复、缩短函数、明确命名、收敛副作用；不能只是换写法。
- 单个业务能力只能有一个事实来源，禁止同一规则在 renderer、API、service 中各写一份。
- 持久化 schema 变更必须写清楚迁移策略；如果可以无损推导，启动时自动迁移。
- 新增跨模块能力优先按 Command/Query/Service 分层接入，不让 Electron bridge 或 renderer 直接承载业务规则。

## 代码规模边界

- `src/electron/renderer/renderer.js` 已是历史大文件；新增功能优先提取纯函数或小模块，避免继续线性膨胀。
- 单个函数超过 80 行时必须先判断能否拆成读取输入、业务计算、渲染输出三段。
- 单个文件超过 500 行且继续增长时，优先按领域拆分，而不是追加工具函数到文件底部。
- 重复出现 2 次的业务规则可先接受；第 3 次必须抽函数或抽模块。
- 正则解析、路径解析、版本发布规则必须集中封装并配测试或集成验证。

## 文档同步

- 功能新增、功能变动、交互文案或配置项变化时，必须同步更新 `docs/usage.md`，让普通用户能按文档完成操作。
- 影响首页介绍、主要功能列表、截图说明或下载启动说明时，必须同步更新 `README.md`。
- 不允许只改代码、测试或内部规范而让用户说明停留在旧行为。

## UI 行为

- 任务日志和订阅日志优先用弹窗展示。
- 任务选择脚本使用树型弹窗，复用脚本管理页的树型心理模型。
- 选择已有脚本时不要展示脚本内容输入框。
- 表格内高频操作使用事件委托，避免每次渲染重新绑定大量节点事件。
- 启用、禁用、置顶等轻量操作优先做乐观更新，失败再回滚。
- 长列表或输入搜索要节流或避免不必要的全量刷新。
- 大列表渲染必须避免无意义 DOM 重建；同样 HTML 不重复写入 `innerHTML`。
- 树型结构必须在构建时缓存统计信息，禁止在每个目录节点递归重复统计。
- 隐藏页面不做轮询重绘；只有当前可见页和打开的日志弹窗允许持续刷新。

## 性能预算

- 普通按钮操作应先本地反馈，再等主进程结果；用户不应该看到“点了没反应”。
- 批量启停、置顶、删除等操作必须优先提供批量接口，避免 N 次 IPC 和 N 次 JSON 写入。
- 日志轮询间隔默认不低于 1 秒；日志文本未变化时不写 DOM。
- 搜索框、脚本树、订阅列表、任务表格的输入联动必须合并渲染或节流。
- 构建性能优化时先减少 DOM 写入、磁盘 IO、进程启动和无界并发，不优先引入新依赖。

## 运行和日志

- `waitForCompletion: false` 时尽快返回 `runId`。
- 前端通过 `getRun` 和 `getRunLog` 轮询实时日志。
- 不在隐藏页面里反复重绘大型列表。
- 日志清理不得删除正在运行的日志。

## 订阅

- 支持 GitHub 仓库、GitHub Raw、普通 HTTP、`ql repo`、`ql raw`。
- 订阅名称优先作为本地脚本目录名。
- 自动建任务只基于明确可解析的 cron 声明，不猜测。
- 自动建任务名称优先取脚本中的 `new Env('任务名')`，其次取 cron 注释 tag，最后取脚本文件名。
- 没有 cron、cron 无效、已存在同脚本任务时跳过。

## 环境变量

- 同名环境变量运行时用 `&` 拼接。
- UI 列表遮罩变量值。
- 测试 CK 只能验证存在性，不能输出完整值到日志或最终回复。
